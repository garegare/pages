    GitHub Issue リンクを正しく生成する Claude Code スキルを作った

    2026年03月11日
    • claude-code
    • github
    • tips

    背景

    Claude Code でブログ作業をしていて、会話の流れを GitHub Issue としてワンタップで起票できると便利だと思い、Issue 作成リンクの生成を試みた。

    ただ単純に実装しようとすると、いくつか問題にぶつかった。

    なお、今回の作業は Claude Code web 版で行っている。ローカルの開発環境はなく gh コマンドも使えない環境なので、PR 作成もリンクを貼る形で対応している。

    問題1: マークダウンリンクでの二重エンコード

    GitHub の Issue 作成 URL は以下のような形式で、title と body をクエリパラメータとして渡せる。

    https://github.com/owner/repo/issues/new?title=タイトル&body=本文

    最初に試したのは、マークダウンのリンク形式でこの URL を書く方法。

    [Issueを作成する](https://github.com/.../issues/new?title=...&body=...)

    ところが、本文に改行（%0A）や ## などの記号を含めると、レンダラーが % を再度エンコードしてしまい、GitHub 側で正しく解釈されなかった。

    問題2: HTML タグはタップで開けない

    次に <a> タグを試したが、Claude Code のチャット UI 上ではタップしてもリンクが開かない。これも実用的ではなかった。

    解決策: 完全 URL エンコード

    title と body のパラメータ値をあらかじめ完全に URL エンコードしておくことで、二重エンコードの問題を回避できた。

    Python3 の urllib.parse.quote を使って、日本語・記号・改行を含むすべての文字を %XX 形式に変換する。

    python3 -c "import urllib.parse; print(urllib.parse.quote('## 概要\n内容をここに書く', safe=''))"
# → %23%23%20%E6%A6%82%E8%A6%81%0A%E5%86%85%E5%AE%B9%E3%82%92%E3%81%93%E3%81%93%E3%81%AB%E6%9B%B8%E3%81%8F

    safe='' を指定することで / なども含めてすべての文字がエンコードされる。また、+ による省略エンコードは使っていない（ブラウザによっては正しく扱われないケースがあるため）。

    生成されるリンクはこういう形になる。

    [新しいIssueを作成する](https://github.com/owner/repo/issues/new?title=%E3%82%BF%E3%82%A4%E3%83%88%E3%83%AB&body=%E6%9C%AC%E6%96%87)

    Claude Code スキルとして登録

    毎回手動でエンコードするのは面倒なので、この手順を Claude Code のスキルとして登録した。

    スキルは ~/.claude/skills/<スキル名>/SKILL.md に置くだけで機能する。

    ~/.claude/skills/github-issue-link/SKILL.md

    内容はこんな感じ。

    ---
name: github-issue-link
description: GitHubのIssue作成リンクを生成する。タイトルと本文を完全URLエンコードしたマークダウンリンクを出力する。
---

# GitHub Issue リンク生成スキル

## ルール
- title と body のパラメータ値は完全URLエンコードする
- 改行は %0A、スペースは %20、コロンは %3A、シャープは %23
- + による省略エンコードは使わない（ブラウザ互換性のため）
- リポジトリURLは git remote get-url origin から取得する

## URLエンコード手順
python3 -c "import urllib.parse; print(urllib.parse.quote('文字列', safe=''))"

## 手順
1. git remote get-url origin でリポジトリURLを取得
2. title・body を python3 でURLエンコード
3. マークダウンリンクを組み立てて出力

    登録後は「IssueリンクをつくってくれやGitHubに報告したい」などと伝えるだけで、Claude が自動的にこのスキルを使ってリンクを生成してくれる。

    まとめ

    方法 結果
    マークダウンリンク（未エンコード） % が二重エンコードされ壊れる
    HTML <a> タグ タップで開けない
    完全 URL エンコード 正常動作

    URL エンコードは一見シンプルに見えるが、レンダラーの挙動によって二重エンコードが起きるケースがある。GitHub Issue のプリフィル URL を扱う際は、urllib.parse.quote で完全にエンコードしておくのが確実。

    Claude Code のスキル機能は ~/.claude/skills/ に SKILL.md を置くだけで有効になるので、手順を定型化したい場面では積極的に使っていきたい。
